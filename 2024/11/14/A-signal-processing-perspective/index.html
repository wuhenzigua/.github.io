A signal processing perspective Federated Learning

Introduction

这里讲了一下机器学习在NLP，CV等方面的成功，通过DNNs算法的成长，但是这个算法是强数据驱动的，很多数据是边缘的，隐私的，所以FL就顺时而生，用于支持边缘学习。然后提出了先进的框架。

从信号处理角度上看联邦学习有三个主要步骤
（1）encoding the local model updates at the edge users into messages conveyed to the server; 在边缘服务器编码成可以传输到中心服务器的数据
（2）transmitting the model updates and the allocation of the channel resources among the users, which takes into account the statistical relationship between the input and the output of the physical communication channel; 在用户间传输模型更新和信道资源分配
（3）combining (postprocessing) at the server. 在服务器组合数据

Basics in federated learning

Federated learning flow

ML are data driven. 模型的建立需要算法（SGD），大量的数据集，训练资源（算力）

Edge users

疑惑是，边缘设备与主服务器之间传递的到底是参数还是梯度

Centralized server

这一段讲了在主服务器里面怎么用边缘设备传递来的参数进行聚合，生成新的全局模型。

Federated learning procedure

每迭代一轮，进行这样的三步操作，
（1），模型分发，
（2），本地训练
（3），全局聚合。
然后不断重复这三步直到收敛。

FedAvg

这里介绍了联邦学习用的一个主要算法，以及加权平均聚合公式

[
\theta_{t+1} = \frac{N}{|\mathcal{G}t|} \sum{i \in \mathcal{G}t} p_i \theta{t+1}^i
]

用于计算下一轮的全局模型参数 ( \theta_{t+1} )。

然后进行参数更新

[
\theta_{t+1}^i =
\begin{cases}
\theta_t^i - \eta_t \nabla f(\theta_t^i; \mathcal{D}_t^i), & \text{if } t \notin \mathcal{I}_T, \
\frac{N}{|\mathcal{G}t|} \sum{j \in \mathcal{G}_t} p_j (\theta_t^j - \eta_t \nabla f(\theta_t^j; \mathcal{D}_t^j)), & \text{if } t \in \mathcal{I}_T.
\end{cases}
]

这里看是不是在聚合步，聚合步进行上传服务器进行聚合，不在聚合步的时候进行本地训练，（使用梯度下降算法。

Fedavg

每个用户在全局聚合和分发之前，使用小批量大小为 B 的 SGD 迭代执行 E > 0 次局部迭代，其中每次更新如下：

[
\theta_{t+1}^i = \theta_t^i - \eta_t \nabla f(\theta_t^i; \mathcal{D}_t^i).
]

其中，( \eta_t > 0 ) 是步长，也称为学习率；( \mathcal{D}_t^i ) 是一个随机采样的小批量，满足 ( |\mathcal{D}_t^i| = B)；( \nabla f(\theta, \mathcal{D}_t^i) \in \mathbb{R}^d ) 是在 ( \theta_t ) 处关于模型参数的目标函数的随机梯度。

通常，用户传递的是本轮生成的模型更新，即 ( \theta_{kE}^i - \theta_{(k-1)E}^i)，而不是模型权重 ( \theta_{kE}^i)。

[
\theta_{t+1} = \frac{N}{|G_t|} \sum_{i \in G_t} p_i \theta_{t+1}^i.
]

我们用的加权平均公式（重要）

本地 SGD 算法的更新规则为：

[
\theta_{t+1}^i =
\begin{cases}
\theta_t^i - \eta_t \nabla f(\theta_t^i; \mathcal{D}_t^i), & \text{如果 } t \notin \mathcal{I}T, \
\frac{N}{|G_t|} \sum{j \in G_t} p_j(\theta_t^j - \eta_t \nabla f(\theta_t^j; \mathcal{D}_t^j)), & \text{如果 } t \in \mathcal{I}_T.
\end{cases}
]

这里，T 是总步骤数，( \mathcal{I}_T \subseteq {1, \ldots, T} ) 表示同步索引集合，即 E 的整数倍的集合。如果 ( \mathcal{I}_T = {1, \ldots, T})，则 E = 1，每次迭代都执行聚合。如果 ( \mathcal{I}_T = {T})，则仅在最后一次执行聚合，即 E = T，这被称为“一次性平均”。
